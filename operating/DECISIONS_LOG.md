## Why this doc exists

Decisions made in chat sessions, even good ones, get lost when sessions end. The HISTORICAL_RECORD docs capture state changes; the DAILY_LOG docs capture work threads. Neither captures *the actual decisions* made and *why* — separated cleanly from the activity that surrounded them.

This log fixes that. Future-Claude reads this first to understand what's been settled. Future-Mike reads this to remember why a thing was decided the way it was.

**The discipline:** every significant decision gets one entry here. Even if it duplicates content in a session log. Especially then.

---

## Entry format

Each decision uses this template:

```
### [YYYY-MM-DD] Short decision name

**Decision:** What was decided, in one sentence.
**Context:** Why this came up; what alternatives were considered.
**Rationale:** Why this option won.
**Reversibility:** Easy to reverse / hard to reverse / one-way door.
**Owner:** Mike / Claude / specific agent.
**Source:** Link to the chat / doc / commit where this was decided.
**Status:** Active / Superseded by [date+name] / Reverted on [date]
```

Keep entries terse. The point is searchability and clarity, not narrative.

## How to use this log

**Adding entries:** when a decision is made in chat that matters, append a row at the top. Use the template. Keep it terse. Even small decisions matter — the discipline is the point.

**Reading the log:** before starting a session, scan the top 10-20 entries to understand recent direction. Before suggesting a tactic, search by topic to see if it's already been decided.

**Reverting decisions:** if a past decision needs reversal, don't delete the entry. Add a new entry titled "[date] Revert [original decision name]" with rationale. Update the original entry's Status to "Reverted on [date]". This preserves the reasoning trail.
